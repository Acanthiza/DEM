3 Methods

3.1 DEMs

DEMs were obtained for two small sample areas in South Australia:

  • Bakara. An area in the Murray Mallee with jumbled dunes, east-west dunes and flat areas
  • Woakwine. An area to the south of the Coorong in the southeast of South Australia. This area rises gently from a lake, to subtle hills through which a trench runs.

Both area have a mix of native vegetation and agricultural land.

3.2 Processing

For each DEM, the following processing was applied:

  • reprojected into GDA 2020
  • masked to the minimum available overlap between all DEMs (in each area)
  • aggregated or disaggregated to roughly 30 m from original resolution
  • reprojected into the same 30 m raster
  • generation of derived rasters via:
    • Qfit function in the MultiscaleDTM package (A. Ilich et al. 2024; A. R. Ilich et al. 2023, 2021) using v argument (window size): 3 cells
    • land_class function in theenvRaster package (Willoughby 2024) using 3 cells for the small window and 9 cells for the large window as input to the 10 class landscape classification
  • sample of the derived rasters 5000 times to generate a range of visualisations and summary statistics.

The Qfit function generates several continuous and one categorical variable for a digital terrain model:

  • elev
  • qslope
  • qaspect
  • qeastness
  • qnorthness
  • profc
  • planc
  • twistc
  • meanc
  • maxc
  • minc
  • features`

A shiny app is available to investigate the meaning of each of these variables.f

3.3 Workflow

The data import, cleaning, analysis and report writing were done in a scripted workflow using the programs R and ‘R-studio Desktop’. R (R Core Team 2024) is an open source software environment for statistical computing and graphics. Base R can be extended via a range of open source packages to enable specific tasks or analyses. The packages used to produce this report are listed in Table 3.2.

R-studio Desktop is a set of open source tools built to facilitate interaction with R.
